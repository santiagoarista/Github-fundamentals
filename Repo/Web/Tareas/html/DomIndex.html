Preguntas

Preguntas

  1. ¿Qué es el DOM?

    Respuesta 1: El DOM (Document Object Model) es una representación en forma de árbol de todos los elementos de una página web. Permite acceder, modificar, eliminar o agregar contenido dinámicamente con JavaScript.

    Respuesta 2 (fuente externa): El DOM es una API para documentos HTML y XML. Representa el documento como una estructura jerárquica que se puede modificar con scripts. [MDN - DOM]

  2. ¿Qué es un nodo?

    Respuesta 1: Un nodo es cada unidad del DOM. Puede representar un elemento HTML, un atributo, un texto, un comentario, etc.

    Respuesta 2 (fuente externa): Un nodo es un objeto que representa una parte del documento. Todos los objetos del DOM heredan de la interfaz Node. [MDN - Node]

  3. ¿Cómo se accede a un elemento del DOM?

    Respuesta 1: Usando métodos como getElementById, getElementsByClassName, querySelector o querySelectorAll.

    Respuesta 2 (fuente externa): El objeto document permite acceder a los elementos del DOM mediante distintos métodos, como querySelector. [MDN - querySelector]

  4. ¿Cómo se modifica un elemento del DOM?

    Respuesta 1: Puedes cambiar su texto con textContent o innerHTML, y sus estilos con element.style.

    Respuesta 2 (fuente externa): Puedes manipular contenido, atributos y estilos de elementos usando propiedades como innerHTML o style. [MDN - Manipulación de documentos]

  5. ¿Cuál es la diferencia entre getElementby* y querySelector*?

    Respuesta 1: getElementBy* accede por ID, nombre de clase o etiqueta, mientras que querySelector usa selectores CSS y es más versátil.

    Respuesta 2 (fuente externa): querySelector devuelve el primer elemento que coincida con un selector CSS; getElementById solo por ID. [MDN - querySelector]

  6. ¿Qué es un evento?

    Respuesta 1: Un evento es cualquier interacción del usuario con la página: clics, teclas, movimientos del mouse, etc.

    Respuesta 2 (fuente externa): Los eventos son acciones detectadas por el navegador, como clicks, cambios de foco, etc., que pueden manejarse con JavaScript. [MDN - Eventos]

  7. ¿Qué es un listener?

    Respuesta 1: Es una función que “escucha” un evento y se ejecuta cuando dicho evento ocurre. Se añade con addEventListener.

    Respuesta 2 (fuente externa): addEventListener() agrega una función que será llamada cada vez que el evento especificado ocurra. [MDN - addEventListener]

  8. ¿Qué es un callback?

    Respuesta 1: Es una función que se pasa como argumento a otra función y que se ejecuta después de que algo sucede, como un evento.

    Respuesta 2 (fuente externa): Una función callback es una función pasada a otra función como argumento, que luego se invoca para completar alguna acción. [MDN - Callback]

  9. ¿Qué es un objeto de evento?

    Respuesta 1: Es el objeto que contiene información del evento que ocurrió, como el tipo de evento, el elemento objetivo, y coordenadas del mouse.

    Respuesta 2 (fuente externa): El objeto Event proporciona información contextual y funcional del evento que se ha activado. [MDN - Event]

Conclusiones

Experiencia general

En general, la experiencia fue bastante sencilla. Pese a que a veces ChatGPT o la IA de preferencia no hace exactamente lo que quieres, la puedes guiar y darle instrucciones más concretas para lograr un mejor resultado

¿Cuándo y para qué usarías ChatGPT y cuándo Copilot?

En realidad siento que ambos pueden darte el resultado que estás buscando con el prompt correcto. A mí en lo personal me gusta usarlos cuando estoy atorado con algún problema y de cierta manera trabajar en equipo con estos modelos para llegar al resultado esperado más rápido y sencillo

Ventajas de usar AI para programar

La principal ventaja es que puedes programar más rápido, resolver problemas de debugging, inspirarte para los diseños, entre otros. Sin duda la ventaja más clara es que el tiempo que te toma desarrollar algo disminuye exponencialmente cuando te apoyas en estas herramientas, siempre y cuando sepas lo que estás haciendo, para así poder corregir sus errores

Desventajas de usar AI para programar

En general considero que existen más ventajas que desventajas, pero una desventaja muy clara es que cuando utilizas IA hay veces que lo que haces no sabes cómo o porqué funciona. Por lo mismo, es importante saber lo que estás haciendo para que así puedas cerciorarte de que lo que hizo la IA es correcto y el mejor acercamiento al problema
